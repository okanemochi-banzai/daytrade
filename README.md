# Daytrade Signal Board

日経225デイトレ・スキャルピング向けの毎朝の寄り前シグナルを自動計算するダッシュボード。
GitHub Actions で毎朝 8:40 JST に実行、GitHub Pages で配信。

## 提供するシグナル

### 1. 朝一先物鞘取りシグナル（最重要）
- 現対（日経先物 − 日経現物）が **±1,000円以上** のハイボラ時に発動
- 現対が +1,000円超 → **鞘取りショート**（先物高 → 先物売りで閉じる方向を狙う）
- 現対が -1,000円超 → **鞘取りロング**（先物安 → 先物買いで閉じる方向を狙う）
- ±1,000円以内 → **ニュートラル**（様子見）

### 2. キリバン値幅（前日終値 ± N円）
- 前日終値に対して ±500 / ±1,000 / ±1,500 円の水準を算出
- これらがレジサポ・天底の目安になりやすい
- **ハイボラ時のみ機能しやすい**（低ボラ時は普通の移動平均で十分）

### 3. 株価キリバン節目（丸い数字）
- 現在価格の上下で最も近い 1,000円単位の水準を算出
- 常にレジサポ候補。軟調相場ほど機能しやすい

### 4. 米日連動判定（前日米国市場 vs 先物）
- ナスダック・SOXの騰落率を表示
- 米強日弱／非連動の検出用

## アーキテクチャ

```
daytrade-signal-board/
├── src/
│   ├── kiriban.py        # 計算コア（純粋関数のみ、I/Oなし）
│   ├── fetch_data.py     # yfinance データ取得
│   ├── dashboard.py      # HTML生成
│   └── main.py           # エントリポイント
├── tests/
│   └── test_kiriban.py   # 計算ロジックのユニットテスト
├── docs/                 # GitHub Pages配信用（自動生成）
│   ├── index.html
│   └── signal.json
├── .github/workflows/
│   └── daily_signal.yml
└── requirements.txt
```

**設計方針**: 計算ロジック (`kiriban.py`) は純粋関数で、データ取得と完全に分離。
これでバックテスト・ユニットテストが書きやすく、将来別のデータソース
（Rakuten / SBI / J-Quants）に切り替えるのも容易。

## ローカル実行

```bash
pip install -r requirements.txt
python src/main.py              # docs/index.html と docs/signal.json を生成
python tests/test_kiriban.py    # テスト
```

## デプロイ

1. GitHubにpushし、**Settings → Pages → Source: GitHub Actions** を選択
2. 平日 8:40 JST に自動でダッシュボード更新
3. `workflow_dispatch` で手動トリガーも可能
